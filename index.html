‚öôÔ∏è 9. Project Methodology & Plan

Part 1: Assess Your Requirements

The requirements of my project are moderately clear with some uncertainty. I have some elements that have known requirements, while others are still in development and I am unsure what will be necessary to complete them.

Part 2: Determine Delivery Cadence

I will continuously deliver small tasks that will add to the final large delivery (the complete portfolio) at the end.

Part 3: Examine Each Approach

Predictive: This method would only work in a small way. It would work to create a plan for my project, but because there are many factors that can change as I go, it would not work for the entire project. The pros are that I could set up a large timeline with expected completion dates that would help motivate me, but the cons are that this method does not promote changes throughout the project, which would be limiting.

Hybrid ‚Äì Incremental: The incremental approach could work for my project as it is mostly made up of smaller tasks that need to be completed independently of each other. This means that I could add components to the portfolio as I went. The challenge with this approach is that I would need to complete the website first in order to add new pieces to it as they are completed, which is not my plan.

Hybrid ‚Äì Iterative: The iterative approach could still work and would be useful for certain parts of my project. For the components, such as certificates, that don‚Äôt need feedback, I would not need this approach, but for the design elements, the about me, and the website creation feedback will be vital to ensuring that the portfolio is the best that it can be. For me, feedback is accessible, because I have stakeholders such as professional contacts and mentors that would be able to help me determine the best course of action for my portfolio.

Agile: The agile approach would be ideal if it was possible, however, the speed of the sprints and the input needed would be more than I want to burden my contacts. I would prefer to have longer timeframes between seeking feedback especially because I may want to alter the components of my project as I work toward completion.

Part 4: Choose Your Methodology

I will use the Hybrid-Incremental approach.

  • Allows flexibility in components of the project.
  • Focuses on completing individual deliverables before moving on.
  • Enables presenting completed pieces for feedback and making adjustments.

Part 5: Project Plan Diagram

I used Lucidchart to create my plan diagram. It reflects my choice of hybrid incremental as it has many small steps that add up to the larger project. I start by working on the design of my website. After getting feedback, I will return to make adjustments, then use the somewhat finalized design to build when writing the HTML/CSS/JavaScript. This step will also go through a round of feedback and editing. Finally, I will adapt my website to ensure mobile responsiveness.

As for the other elements of my project (certificates, projects, and various content), these are less dependent on each other. I will complete these individual pieces throughout the semester and keep them ready to be added to the final portfolio once the website is complete.

Part 6: Tools

Tools Considered: Draw.io, PowerPoint, Lucidchart

I chose to use Lucidchart. It balances freedom and structure. Compared to others:

  • Draw.io: Flexible but unfamiliar mechanics for my approach.
  • PowerPoint: Structured but lacked freedom to adapt my vision.
  • Lucidchart: Provided templates, sticky notes, and tables to match my project thinking. Free version was sufficient.

Lucidchart allowed me to adapt templates and create my plan diagram efficiently.

Part 7: Project Phases

My project is set up in phases, during which I will be working on various tasks that are either completely or somewhat independent of each other. Each of these tasks will eventually add up to become the finished portfolio.

During the planning phase, I will focus on researching for my certificates, design, and smaller projects, with a quick feedback review before beginning the execution phase. The execution phase will be the longest of my project during which time I will be working on different deliverables for my project at different times. Most of my time spent during this phase will be in completing the smaller steps toward the final project with a feedback session after the completion of each step. Any adjustments necessary after feedback will occur in the week after receiving the feedback. In the later phase, testing, I will focus on getting all of the material/content into the website and I will also ensure that everything is working as I intended. Finally, during the deliver phase, this is when I turn in the project.

Setting up the tool went as I had hoped, but I was pleasantly surprised that there were templates that helped me achieve exactly what I wanted with the final outcome. It was, however, harder to use the tool in certain ways, such as getting the columns to look as I wanted them to, and it took some time to learn the best ways to work with Lucidchart.
